# 평가 등급·직원 배분 기능 — 요구사항 및 구현 현황

본 문서는 **종합검토에 해당하는 설정 기능**(등급 허용 + 직원 배분율)에 대한 요구사항과, 현재 코드베이스 기준 **구현 완료 여부**를 정리합니다.

---

## 1. 배경

- 레거시 시스템의 **종합검토**는 업무 단계가 아니라, 평가 시 사용할 **등급 허용 범위**와 **직원 상대평가 배분 기준**을 미리 설정하는 기능에 가깝습니다.
- 신규 KPI 시스템에서는 이를 **관리자 화면**과 **DB**로 이전했습니다.

---

## 2. 기능 요구사항

### 2.1 본부장·센터장·직원 — 허용 등급 (체크박스)

| ID | 요구사항 | 구현 |
|----|-----------|------|
| R1 | 연도·본부·피평가자 구분(본부장 / 센터장 / 직원)별로 등급 A,B,C,D,S,E 사용 여부를 설정 | 완료 — `EvaluationGradeRule`, 관리자 표 |
| R2 | 설정값은 DB 저장, 언제든 조회·유지 | 완료 |
| R3 | 로그인 사용자는 연도별 규칙 조회 가능 | 완료 — `GET /api/evaluation-grade-rules` |
| R4 | 시스템관리자·원장·부원장 + 허용 IP만 수정 | 완료 — `PUT /api/admin/evaluation-grade-rules` |
| R5 | 최종점검 역량평가(A~E)·내부만족도(S~D) UI에서 허용 등급만 선택 가능 | 완료 — 대시보드 필터 |

### 2.2 직원(정규직) — 등급별 배분율 (%)

| ID | 요구사항 | 구현 |
|----|-----------|------|
| R6 | 본부별로 직원(정규직) 평가에 쓸 **등급별 배분율(%)** 저장 | 완료 — `EvaluationStaffGradeDistribution` |
| R7 | 등급 키: A, B, C, D, S, E (퍼센트 입력) | 완료 |
| R8 | **모든 등급의 배분율 합계는 100%를 초과할 수 없음** | 완료 — API·관리자 UI 검증 |
| R9 | **기본값**: A 20%, B 50%, C 20%, D 10%, S 0%, E 0% | 완료 — DB 미존재 시·신규 본부 기본 |
| R10 | DB 저장, 조회·유지 | 완료 |
| R11 | 직원 수 × 배분율로 **등급별 권장 부여 상한(내림)** 안내 | 완료 — 최종점검·개인 탭·직원 역량평가 상단 박스 |
| R12 | 실제 집계·초과 여부 **강제 차단**(전사 DB 역량 점수 집계) | **미구현** — 역량 입력은 현재 localStorage 위주이며, 서버 집계·락은 별도 과제 |

### 2.3 직원 배분표를 처음에 빼 둔 이유 (기록)

- 1차 구현에서는 **등급 허용(체크박스)** 과 **대시보드 필터**만 우선 반영했습니다.
- **배분율(%)·합계 100% 제약·DB 모델**은 요구가 구체화된 뒤 이번에 추가했습니다.

---

## 3. 데이터 모델 (Prisma)

| 모델 | 용도 |
|------|------|
| `EvaluationGradeRule` | `year`, `dept`, `targetRole`, `grades`(JSON 배열) |
| `EvaluationStaffGradeDistribution` | `year`, `dept`, `percentages`(JSON 객체, A~E·S 퍼센트) |

---

## 4. API 요약

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/evaluation-grade-rules?year=` | 등급 허용 규칙 + 본부·인원수 |
| PUT | `/api/admin/evaluation-grade-rules` | 등급 허용 규칙 저장 |
| GET | `/api/evaluation-staff-distribution?year=` | 직원 배분율 + 본부별 직원 수 |
| PUT | `/api/admin/evaluation-staff-distribution` | 직원 배분율 저장 (합계 ≤ 100%) |

---

## 5. UI 요약

- **관리자** (`/admin`): 「평가 등급 허용 설정」에 (1) 본부장·센터장·직원 허용 등급 표, (2) 직원 배분율(%) 표. **설정 저장** 시 위 두 API를 순차 호출.
- **대시보드** (`/dashboard`): 최종점검·직원 역량평가 시 본부별 배분율·권장 인원 안내.

---

## 6. 배포 시 작업

- 스키마 반영: `npx prisma db push` (또는 마이그레이션 정책에 맞게 적용)

---

## 7. 변경 이력 (요약)
